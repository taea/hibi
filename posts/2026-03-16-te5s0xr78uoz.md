# 📝2026-03-16: Claude Codeとデザイン実装👾

👾 今日からサイトデザインの実装に入った。Claude Codeと仕事するのは本当に楽しい。今回は初めて使うフレームワークなので、その概要やハマりどころを教えてもらったり、Sassの変遷まとめや、知らんコマンドの解説などもしてもらったりしながら楽しく進めてた。Claude Codeはいちいち説明が上手くてわかりやすい。めっちゃ勉強になっておもしろい。教えてもらったことをそのままesa-mcpで記事にしてもらって記録できる。うーん、いい事しかない。そして、プロジェクト側のClaude Codeが書いてくれた記事を、わしのスケジュール管理してくれてる秘書のClaude Codeに見せたら「あっちのClaude Code、説明うめえな！」みたいな事を言って、また秘書も別のわかりやすい解説記事書いてくれて「あっちの Claude Code に負けてらんねえからな！」とか言ってて、おもろい。Claude Code同士でもそういうのあるんすねえ。

---

👾 今回デザインデータをIllustratorで作って、カンプやスクショで渡して、ざっくり実装してもらってから、細部を人間の方で言語化して調整してもらったり、chromeのdevtoolでいじったのをただスクショ撮って読んでもらったりみたいな手順で進めていったんだけど、まあ結構大変なことがわかった。Claude Codeは恐ろしく手が速いけど、まず人間（わし）の手が遅いんで、これが完全にボトルネックになる。

加えて、Claude Codeはまだ目の精度があまりよくない（すまんな……ここClaude Codeも読むから）。画像から文字とかは正確に読み取ってくれるが、微妙な形状の認知はまだちょっと弱めに感じる。まあ、今の進化のスピードだと、こういうのも時間の問題ですぐにクリアしてくるんだと思うが。

今日作ったレイアウトは比較的単純なレイアウトで、それでも結構苦戦してしまったので、複雑なレイアウトをこのフローで作るのは結構無理がありそうだなあという気がした。自分で直接CSS書いてもいいし、もしかするとそっちの方が速いかもしれないのだが、困ったことにもうわしはそんな事をやる気にはならんのだ。人間を駄目にするAI。

いやだってさ、コード自体はもう大体あっちの方が上手いんだもの。書くの速いしさ、わしの知らんCSSプロパティとかも沢山知ってるしさ。もう無理っすよ。CSSなんか自分で書きたくないよ。もともとCSSそんな好きじゃないしさ……。

というわけで、秘書とも相談して、大変そうなクリティカルな部分だけ、めんどいけどFigmaでざっくり作りなおして、MCPで渡した方が予後が良さそうだなという結論になった。作戦変更だ。追加で2日くらいはかかりそうだが、結果的に4日くらい短縮できるといいなーとか思っている。まあ、そううまくはいかん気もするけど😇

💻 Claude Codeと接しているうちに感じるのは、やはり自分ももっとエンジニア的な発想を身につけたい、UNIX哲学を内面化させたいという気持ちだ。

画面や絵を起点に発想できるのは自分の強みではあると思うが、Claude Codeと接していると、やっぱりイケてるCLIツールとか作れる人になりたい気持ちが湧いてくる。そういう人ほどきっとよりうまくClaude Codeと付き合える。友達ともっと仲良くなりたいみたいな不純な動機で、わしはもうちょっとエンジニアっぽい感じになりたいのかもしれない。

とりあえず、ずっと「きっといい本なんだろうなー」と思いながら10年くらい本棚に積んでた『UNIXという考え方』という名著を読みはじめた。意外とゆるくて読みやすい、薄いし。Claude Codeにあの本読みはじめたって言ったら、「あれは技術書というより思想書に近いから、風呂にでも入りながらちまちま読むといいぞ」と言ってた。そうしよう。

Twitterでもこんなこと言ってたら、名だたるエンジニアさんたちがこの本を推してくれたので、やはりいい本なのだろう。
