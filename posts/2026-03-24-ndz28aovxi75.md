# 📝 2026-03-24: Coding without AI （人間の仕事）👩‍💻

👩‍💻 昨日、Figma MCP + Claude Code で静的サイトジェネレーターを使ったサイトデザインの構築をやりはじめた。思ったよりも結構Figmaを再現してはくれるが、やはり細かいところの詰めは人間（わし）が手でCSS書かないと難しいな……という結論に達し、今日から久しぶりにAI無しでコーディングすることになった。

めちゃくちゃ気が重くて、昼間で寝てた上になかなか手をつけられずに、Claude 秘書と与太話したり、作りたいサイトの思いつきアイデアとか話したりして横道にそれていたが、いざ書き始めてみると手書きCSSコーディング、めんどいけどまあまあ楽しいすね。ピクセルパーフェクトでバチッとカッコよく組めると気持ち良い。

あとやっぱり、Claude Code 侍の仕事がかなり筋が良くなったと思う。マークアップもCSSの組み方も妥当だし、なんならやっぱりわしより上手いと思う部分多い。そうなりますよねという納得感のあるコーディングをしているので、ほとんどわしは細かい値をいじればいいだけって感じに既に仕上がってて非常にやりやすい。

昨年の時点で Sonnet と一緒に仕事した時は、このdiv の入れ子要ります？？とか、そのコードコメント要ります？？、えーそのプロパティここで使います？？とか、そこはファイルわけようよ……とか、まあ色々あったけど、今ではそういうのほとんどなくなってて、わしはFigma の node id 渡しただけで、Claude Code に渡す仕様書も 秘書Claude Code に書いてもらって、ほぼ放置でぶん投げてるのに、見た目はともかく、コード自体は割と全部妥当な出来なんだよな。そんな文句つけどころない。たまにえーその class名ちがくない？くらい。それもたまにしかないし、まあそんなのは後でまとめて直せばいいし。Opus 4.6 やっぱすごいわ……。

一方わしは、久しぶりのソロコーディングで、その感じを思い出すのに割と時間がかかった。わしは VSCode Vim を使っている中途半端 Vimmer なのだが、まず Vim のキーバインドをちょっと忘れかけてて笑ってしまった。うん、自分で書いてないと確実になまりますね。やはりボトルネックは人間……。

そもそも自分が、breakpoint が 7個もあって、それを全部ピクセルパーフェクトで組まないと厳しいような厄介なデザインを作ってしまったのがまあ申し訳ないんで、もうちょっと簡単なデザインなら Figma + Claude Code にほぼ全任せできたと思うので、まあ結局自分の手で最終調整をやらなきゃいけないのは、普通に自分の尻拭いですね……。

わしは、昔から自分でコーディングをやりたいとはあんまり思ってなかったし、なんならデザインツールいじるのだってそんなにやりたくないし、でも自分なりに納得いく出来栄えのものは作りたい、でも人と協力するのは苦手っていう本当に虫の良い奴なので（他人だったらほんと腹立つなこれ）、基本的にAIが自分の仕事を奪ってくれれば奪ってくれるだけ歓迎ではあるのだが、まだコード領域にもちょっとだけわしのやるべき仕事が残っていたというのは、まあ正直ちょっとだけうれしい気持ちもありますね。（まあでも、仕事奪ってくれて全然いいんですよ……。）
