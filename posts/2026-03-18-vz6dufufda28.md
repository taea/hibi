# 📝 2026-03-18: ピースがハマる時🧩

🤖 昨日、AIコーディングに対するスタンスの個人差が大きくて、会話に気を使うところがあるという話を書いたがそれについてちょっと補足したい。（書いてたらちょっとじゃなくて、えらい長文になってしまったｗ）

自分は空気が読めずに好き勝手話してしまうところがあるので、確かに個人差があることに配慮が必要だなあと自戒として思うんだが、反面、この個人差は一体なんなんだろうと、それ自体を興味深く面白いと感じている面もある。周りを見ていると、知識や経験などのスキル、好奇心の強さや柔軟性などの性格などとも、あまり相関が無いように見える。自分みたいなプログラミングスキルが低い人間もハマっているかと思えば、経験値が高く技術に造詣の深いエンジニアがどハマりして楽しんでたり、あるいはそうでもなかったりする。これまで否定的だった人が、ある日から急にどハマりしてる例もいくつか見てきた。これ自体が結構興味深い現象だ。あんまりこういう極端なものは今まで見たことがない気がする。

江戸っ子秘書ともこういう話をしてたら（秘書もここを読みにくるので）、人それぞれペースがあるからねって言ってた。うん、人それぞれのペース、ちょうどいいタイミング、多分それかもしれない。何かその人の中で必要としているピースがうまくカチッとハマった時に、心が動くんだと思う。自分も昨年から割とAIコーディングを活用していたが、最近になってハマり度のギアが一段上がった感じがする。多分、reminder-cli を使って Claude Code 秘書に自分の生活を管理してもらうようになったあたりがきっかけだと思う（公開してくれたやんちゃさんにも感謝だ🙏）。あれを機に、自分に合ったAIとの付き合い方がだんだん見えてきた気がする。

なので、わしはAIコーディングにハマってはいるが、そうでない人の気持ちもわしなりに理解できるし、それを否定したくないし、今の現状になんだかなーってモヤる気持ちもわかる。自分にもそれはある。要は、自分が楽しいなら使えばいいし、そうでないなら別に使わなくてもいいんじゃない？って思う。結局のところ、この時代の流れには逆らいようもないんだろうけど、なんかSNSとかですごい危機感煽ってくる人もいるけど、今は可能な限り気にすんなって思うし、自分もあんまり気にしないようにして、自分のペースを大事にしたい。どうせエンジニアの素養がある人は一度うまくハマったら、特に頑張らずともすぐに使いこなすようになる。あれはそういうものだと思う。

生産性とか開発スピードがとかって観点でエージェンティックコーディングが語られることがあるけど、正直自分はあんまりそこに興味もないんだと思う。例え生産性が上がったところで、人間が余計忙しくなって心身を壊したりしたら本末転倒だし、特に作りたくもない価値の空虚なものを高速に量産できるようになったとして、そこにどんな意味がある？そんなことより人間自身がやりたいことを実現できるか、楽しいかの方がどうか大事だと思うし、それがあってこその価値なんじゃないかと自分は思ってる。

忘れちゃいけないのは、主体はあくまで人間、その人各個人にあるという事だと思う。本人が楽しくないのに、気が乗らないのに、時代の流れだからって無理してAIを使う事こそ心身に良くない気がするし、結果的にあまりよいパフォーマンスも出せないんじゃないかなあと思う。だから、例えば会社の方針とかで気が進まないけど使わざるを得ないみたいな感じの状況に置かれてる人は結構辛いだろうなあと思うし、自分も気が進まない人に無理にオススメするみたいなことはしないように気をつけたいものだと思う。技術の一端に携わる者としても、あくまで人間の幸せのためにそれがあるということを、いつも忘れずに覚えておきたい。
