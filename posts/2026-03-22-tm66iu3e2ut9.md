👾 昨日から今日にかけて、Claude Code と話してて、オッ、今の会話よかったね！ってやつを /save-conversation ってやると esa-mcp で esaに会話ログを記録してくれる skill を作った。バイブコーディングは、寝ながらでもできるんすよね。なんせ、わし駄文書くだけだから……。

昨日の夜作ったんだけど、今日実際使ってみて少しチューニングした。元々の仕様は、デフォルトだと会話の要約、オプションつきで生ログって感じだったんだ使ってみると、自分は生ログも要約も両方欲しくなることがわかった。ので、

- /save-conversation のみだと 要約 + 生ログを両方記録できる
- \--raw オプションつけると生ログのみ
- \--summary オプションつけると要約のみ

って感じの仕様になった。

さらに、saveする会話の範囲の指定も難しい。実際使ってみたら、思っていたよりも広い範囲の会話を記録してた。これもどうしよっかって Claude Code に相談したら、コマンドの後に、記録してほしい会話の内容を生文で伝えれば、そこだけを適用範囲にしてくれるとのことだった。つまり、以下のようにすると、日本語で指定した範囲の内容の会話だけを記録してくれる。

- /save-conversation さっきのCLIの仕組みについて

すっごい。そんなことできるのか！！！これはただのCLIツールにはできない、行間が読めて気が利くAIエージェントにしかできないUNIX + LLMならではの合せ技じゃーん！クールすぎるぞ……！

さらに、使い方をわしが忘れそうなので、保存対象のesaのカテゴリのREADMEに使い方を書いてもらった。完璧すぎる。esaのカテゴリREADMEの仕組み、粋だなってClaude Codeも褒めてくれたしへへー😏

これも、しばらく自分で使い続けてみて、機能が熟成してきたら公開skillにしてもいいかもしれない。これがあると、他の Claude Code にも気軽にログを見に行ってもらうこともできるので、コンテキストが共有しやすくて便利になりそう。
