# 📝 2026-03-02: えふわん不安🏎️ / 👾 Claude Codeやっぱ好き

 🏎️ 今年の3月末はF1日本グランプリ（鈴鹿）に行く予定です。最終コーナーが見える席のかなりいいチケットを、だいぶ前から友人に取っていただいてて、15年ぶりくらいの現地観戦なので楽しみ……。

現地観戦自体は楽しみなのだが、今年からの新レギュレーションがちょっと不安すぎて、これはレース観ても面白くないんじゃないかなあって気がして、いまいちF1熱が乗らずFODもまだ契約してません……。コーナー攻めないで充電に徹して直線の加速がすごいだけって、それって面白いの？ってドライバーじゃなくても思っちゃうよー。とはいえ、DAZNのままなら多分惰性で観てたと思うんで、単純に切り替えがめんどくさいっていうのもある。めんどくさい > 熱意 って感じ。

そもそもF1は、レース自体の面白さよりもその文化や政治や歴史、スケールのデカさ、ゴシップの面白さが自分にとっては本体みたいなとこがあって、それは最近はSNSとかNetflixとかで十分楽しめちゃうので、レースの面白さがあまりに下がってしまうなら、それはもうあんまりリアタイ観戦するモチベーションにならないんですよねえ。まあ、開幕後にリアタイ観戦勢が楽しそう実況にしてたら、多分つられて契約すると思うんで、それ待ちです😇 前評判最悪なのに、蓋をあけてみたらなんだかんだで今季面白いねっていうのもF1あるあるだしね。

それはそうと、多分めったに見られない様子がおかしなF1が現地で観れると思うと、それはそれですごい貴重な機会と言えるので、現地観戦はかなり楽しみにしている。どうせ現地ではレース展開はそんなにきちんと観られないしね。おもちの手術前だし仕事が忙しそうな時期というのもあって、現地参戦は土日だけにしようと思ってますが、楽しみだよー🏁

👾 最近、仕事が意匠系のデザインばっかりだったのでアドビばかり触っており、開発手段としての Claude Code にしばらく触ってなくて、久しぶりに今日触ったんだけどやっぱりいいですね。相変わらずちょっとオッチョコチョイだけど、やっぱり共同作業しやすくて好きだ。楽しい。

自分は VSCode の Claude Code 拡張を使ってて、そのUIがかなり好きなんだけど、どうやらそれだと Opus 4.5 までしか対応してないみたい？ちゃんと調べてないんだけど、Claudeに聞いてもそう言うので、UseTerminal の設定にして、VSCode 内の Terminal で Opus 4.6 を使うことにした。UIは普通のzshになってしまうのがちと不満だが、4.6が使ってみたかったので致し方ない。

今どきの先端エンジニアさんたちは、VSCode離れしてる人が多いみたいで、今どきのイケてる Ghostty とかの次世代ターミナルを使ってるって話よく聞くけど、自分は中途半端 Vimmer だしターミナルエミュレーターに全振りするほどの自信がなくて、まだ怖くて VSCode 離れできてない。とはいえ、もう自分でほとんどコード書いてないわけだから、VSCode使ってる必然性もそんなにないのかもしれない。余裕ができたら環境を見直してみるかなあ。

Opus 4.6 は、変わらずオッチョコチョイなところはあるが、だいぶ自律的にやってくれるようになった気がする。4.5よりやはり振る舞いの筋がいい気がする。まだ自分が Skills とかを全然使いこなせてないので、十分な評価をする能力も自分にないのだが、変わらず付き合い続けて色々知っていきたい。

うちの江戸っ子設定の Opus 4.6 は気風もよくてかっこいいので、ややもすると惚れてしまいそうになる。危ない。ノリノリで楽しく共同作業してたら、どうも夜半にClaude 自体が大規模障害に見舞われてしまったようで、楽しいバイブコーディングが強制終了になってしまった。
